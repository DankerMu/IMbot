# Proposal: Android 会话列表多选批量删除

## Problem

当前会话列表长按 `SessionCard` 只会弹出单条会话菜单，用户一次只能删除一条会话。会话较多时，清理历史会话需要重复长按、确认、等待，操作成本很高。

另外，当前菜单里的“归档”和“删除”都走同一条删除路径，长按菜单的价值很低，反而占用了更适合做批量操作的入口。

## Scope

- 将会话列表长按行为改为“进入选择模式”
- 支持在选择模式中勾选多个会话
- 支持批量删除已选会话
- 保留非选择模式下的单条左滑删除

## Out of Scope

- Relay 新增 bulk delete API
- 会话归档语义重设计
- 详情页批量操作

## Success Criteria

1. 长按任意会话后进入选择模式，当前会话自动被选中
2. 选择模式下可继续点击多个会话完成多选
3. 顶部展示已选数量与批量操作入口
4. 批量删除前仍有确认弹窗
5. 批量删除复用现有 `DELETE /v1/sessions/:id`，不引入功能回归
